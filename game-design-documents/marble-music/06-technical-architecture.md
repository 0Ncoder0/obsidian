# 06 -- 技术架构与模块职责

## 1. 总体架构

前端单体应用，无后端依赖。所有逻辑在浏览器内运行。

```
┌─────────────────────────────────────────────────────────┐
│                       GameApp                           │
│                  (编排入口 / 主循环)                     │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │   app/   │  │  scene/  │  │ physics/ │  │  audio/ │ │
│  │ 状态编排 │  │ 实体模型 │  │ 物理引擎 │  │ 音频系统│ │
│  │ 模式控制 │  │ 编辑操作 │  │ 预测模拟 │  │ 可视化  │ │
│  │ 输入路由 │  │          │  │ 相机跟随 │  │         │ │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬────┘ │
│       │             │             │              │      │
│  ┌────┴─────┐  ┌────┴──────────┐                       │
│  │   ui/    │  │ persistence/  │                       │
│  │ 面板渲染 │  │ 序列化/保存   │                       │
│  │ HUD/五线谱│ │ 本地存储      │                       │
│  └──────────┘  └───────────────┘                       │
└─────────────────────────────────────────────────────────┘
```

### 技术栈

| 技术            | 用途                     |
| --------------- | ------------------------ |
| Vite            | 构建工具、开发服务器     |
| TypeScript      | 类型安全的应用逻辑       |
| Matter.js       | 2D 物理引擎             |
| Web Audio API   | 音频合成与播放           |
| Canvas 2D       | 场景渲染                 |

## 2. 模块职责

### `app/` -- 应用状态编排

| 类                   | 职责                                                       |
| -------------------- | ---------------------------------------------------------- |
| `GameApp`            | 应用入口，初始化所有子系统，驱动主循环，处理跨系统事件     |
| `ModeController`     | 管理编辑/播放状态机，控制权限闸门（谁能做什么）            |
| `InputController`    | 统一捕获键鼠事件，根据当前模式分发到对应处理器             |

### `scene/` -- 实体与场景数据

| 类                   | 职责                                                       |
| -------------------- | ---------------------------------------------------------- |
| `SceneManager`       | 管理实体集合（增删改查），维护场景状态，场景变更时通知预测系统 |
| `EntityFactory`      | 创建 Ball / Block / MusicBlock 实例                        |

### `physics/` -- 物理模拟与预测

| 类                        | 职责                                                   |
| ------------------------- | ------------------------------------------------------ |
| `PhysicsWorld`            | Matter.js 封装：实体与刚体映射、步进模拟、碰撞事件输出 |
| `PredictionEngine`        | 基于场景快照运行预测模拟（克隆世界 + 模拟 N 步），输出预测轨迹和预测碰撞事件（PredictedNote），供预测线和 Timeline 五线谱共同使用 |
| `CameraFollowController`  | 管理自动跟随与手动相机的协调                           |

### `audio/` -- 音频系统

| 类                 | 职责                                                       |
| ------------------ | ---------------------------------------------------------- |
| `AudioEngine`      | 接收碰撞触发，调度 voice 创建/生命周期/混音输出，执行限流  |
| `PianoSynth`       | 音名到频率映射，配置音量驱动的衰减包络，生成单次 voice 实例 |

### `ui/` -- 界面渲染

| 类                       | 职责                                                       |
| ------------------------ | ---------------------------------------------------------- |
| `PanelRenderer`          | 右侧积木选择面板和参数编辑面板的渲染与交互                 |
| `TimelineStaffRenderer`  | 底部 Timeline 五线谱渲染：读取 PredictedNote 数据，按小球分组绘制五线谱 |
| `HudRenderer`            | 播放状态和保存状态的 HUD 覆盖层                            |
| `CanvasRenderer`         | 主画布渲染：网格、实体、选中高亮、预测线、可视化特效       |

### `persistence/` -- 持久化

| 类                   | 职责                                                       |
| -------------------- | ---------------------------------------------------------- |
| `LocalSaveRepository`| 节流保存（throttle）和恢复，封装 localStorage 读写         |
| `SceneSerializer`    | 场景数据的序列化（对象 -> JSON）和反序列化（JSON -> 对象） |

## 3. 主循环

`GameApp` 通过 `requestAnimationFrame` 驱动主循环。每帧按以下顺序执行：

```
┌─ 每帧执行顺序 ──────────────────────────────────────┐
│                                                      │
│  1. 输入处理                                         │
│     └─ InputController 拉取事件，按模式分发           │
│                                                      │
│  2. 模式状态更新                                     │
│     └─ ModeController 处理模式切换请求               │
│                                                      │
│  3. 预测更新（仅编辑态，且场景有变更时）             │
│     └─ PredictionEngine 重新模拟                     │
│        ├─ 输出预测轨迹 → 预测线渲染                  │
│        └─ 输出 PredictedNote[] → Timeline 五线谱渲染 │
│                                                      │
│  4. 物理步进（仅播放态）                             │
│     ├─ PhysicsWorld.step()                           │
│     └─ 收集实时碰撞事件                              │
│                                                      │
│  5. 音频触发（仅播放态）                             │
│     ├─ AudioEngine 处理碰撞触发                     │
│     └─ 更新可视化数据                                │
│                                                      │
│  6. 相机更新                                         │
│     └─ CameraFollowController（自动跟随或手动）      │
│                                                      │
│  7. 渲染                                             │
│     ├─ CanvasRenderer 绘制画布（含预测线）           │
│     └─ UI 组件更新（面板、HUD、Timeline 五线谱）     │
│                                                      │
│  8. 自动保存检查                                     │
│     └─ LocalSaveRepository（节流条件满足时写入）     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**关键约束**：
- 物理步进使用固定时间步长（fixed timestep），与渲染帧率解耦
- 预测计算仅在编辑态且场景发生变更时触发，避免每帧重复计算

## 4. 状态机与模式切换

### 状态定义

```
  ┌────────┐   Space    ┌────────┐
  │  Edit  │ ─────────► │  Play  │
  │        │ ◄───────── │        │
  └────────┘  Space/Esc └────────┘
```

仅两种状态，无中间态或暂停态。

### 切换时的系统行为

| 步骤 | Edit -> Play                          | Play -> Edit                          |
| ---- | ------------------------------------- | ------------------------------------- |
| 1    | 锁定编辑权限                          | 停止物理步进                          |
| 2    | 隐藏右侧面板 + 底部 Timeline 五线谱  | 停止音频触发（已发声允许自然衰减）    |
| 3    | 隐藏预测线                            | 恢复编辑权限                          |
| 4    | 停止预测计算                          | 显示右侧面板                          |
| 5    | 判定跟随目标（选中球？）              | 触发预测重算，恢复 Timeline 五线谱和预测线 |
| 6    | 启动 PhysicsWorld 步进                | 触发强制保存                          |
| 7    | 启动 AudioEngine 监听                 |                                       |

## 5. 模块间数据流

### 编辑态数据流（预测）

```
场景变更
SceneManager ──场景快照──► PredictionEngine
                                │
                                ├── 预测轨迹 ──► CanvasRenderer（预测线）
                                │
                                └── PredictedNote[] ──► TimelineStaffRenderer（五线谱）
```

### 播放态数据流（实时）

```
碰撞检测                     音频系统                   可视化
PhysicsWorld ──碰撞事件──►  AudioEngine ──活跃voice──► CanvasRenderer
```

### 通用数据流

```
用户输入                     场景管理                   持久化
InputController ──操作──►  SceneManager ──场景数据──► LocalSaveRepository
                                │
                                ├──实体数据──► PhysicsWorld（刚体同步）
                                └──实体数据──► CanvasRenderer（渲染）
```

## 6. 性能与稳定性约束

| 约束                     | 说明                                                   |
| ------------------------ | ------------------------------------------------------ |
| 固定物理步长             | 物理步进与渲染分离，减少帧率波动对模拟结果的影响       |
| 预测惰性计算             | 预测仅在编辑态且场景有变更时触发，播放态完全跳过       |
| 预测节流                 | 高频编辑操作时预测计算需节流（如 100~200ms 去抖），避免拖拽中每帧都重算 |
| 存档节流                 | 自动保存有最小间隔（如 1~2 秒），避免高频写入阻塞主线程|
| voice 回收               | 衰减结束的 voice 及时销毁 AudioNode，防止内存泄漏      |

| 观测项           | 说明                                   | 显示条件     |
| ---------------- | -------------------------------------- | ------------ |
| FPS 计数器       | 当前帧率                               | 仅开发模式   |
| 活跃 voice 数    | 当前正在发声的音符实例数量             | 仅开发模式   |
| 每秒事件量       | 每秒碰撞触发次数                       | 仅开发模式   |
| 预测耗时         | 单次预测模拟的计算时间                 | 仅开发模式   |
| 预测音符数       | 当前预测结果中的音符总数               | 仅开发模式   |
| 模式切换日志     | 记录切换时间和方向                     | 始终          |
| 存档失败日志     | 记录失败原因和数据快照                 | 始终          |
| 音频上下文状态   | AudioContext 状态变化（suspended/running）| 始终       |

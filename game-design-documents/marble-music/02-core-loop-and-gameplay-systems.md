# 02 -- 核心循环与玩法系统设计

## 1. 核心游戏循环

玩家的体验围绕一个不断迭代的循环展开：

```
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  ▼                                                          │
搭建场景 ──> 配置音乐参数 ──> 预览五线谱 ──> 播放验证 ──> 调整优化
 (编辑态)      (编辑态)        (编辑态)      (播放态)     (编辑态)
```

### 各阶段详细说明

| 阶段             | 模式   | 玩家行为                                                         |
| ---------------- | ------ | ---------------------------------------------------------------- |
| 1. 搭建场景      | 编辑   | 从右侧面板选择积木，在画布上放置和拖拽排列                        |
| 2. 配置音乐参数  | 编辑   | 选中音乐方块，调整音名、音量                                      |
| 3. 预览五线谱    | 编辑   | 观察 Timeline 五线谱中预测的音符序列，判断布局是否符合预期         |
| 4. 播放验证      | 播放   | 按 `Space` 启动，观察真实物理运动和碰撞发声，验证预测是否准确     |
| 5. 调整优化      | 编辑   | 按 `Space` 或 `Esc` 停止，根据播放体验调整积木位置和参数，进入下一轮 |

**与旧设计的核心区别**：Timeline 不再是"播放后回看记录"，而是编辑态的**实时预测工具**。玩家在搭建和调整的过程中就能通过五线谱"看到"即将产生的音乐，无需等到播放后才能理解结果。

## 2. 模式系统

游戏在任何时刻处于且仅处于两种模式之一。不存在暂停态或其他中间状态。

### 编辑模式（Edit）

玩家拥有完整的场景操作权限：

- 新增积木：从右侧面板选择类型，点击画布放置
- 选中积木：点击画布上的实体
- 移动积木：拖拽已选中的实体
- 删除积木：选中后按 `Delete` 或 `Backspace`
- 配置参数：选中音乐方块后，在右侧面板编辑音名和音量
- 查看 Timeline 五线谱：底部区域实时显示预测的碰撞音符序列
- 查看预测线：小球的预测运动轨迹可见

**Timeline 与预测线的关系**：两者基于同一套预测模拟引擎。预测线展示小球"去哪里"，Timeline 五线谱展示"会触发什么音"。场景任何变更（放置/移动/删除积木、修改参数）都会触发两者同步更新。

### 播放模式（Play）

场景完全锁定，物理模拟和音频系统接管：

- 所有编辑操作被禁止（新增、移动、删除、参数修改）
- 右侧编辑面板和底部 Timeline 五线谱隐藏
- 预测线隐藏
- 物理引擎驱动小球运动
- 碰撞事件触发音符发声
- 用户仅可进行视窗操作：平移和缩放

### 模式切换规则

```
              Space
  编辑模式 ◄────────► 播放模式
              Space
              Esc ──► 编辑模式（单向）
```

| 触发     | 当前模式 | 目标模式 | 附加行为                                              |
| -------- | -------- | -------- | ----------------------------------------------------- |
| `Space`  | 编辑     | 播放     | 锁编辑、隐藏 UI（含 Timeline）、判定跟随目标、启动物理/音频 |
| `Space`  | 播放     | 编辑     | 停止物理/音频、恢复 UI（含 Timeline 预测刷新）         |
| `Esc`    | 播放     | 编辑     | 同上                                                  |
| `Esc`    | 编辑     | --       | 取消当前临时放置态（若存在），不切换模式               |

## 3. 实体系统

### 3.1 小球（Ball）

| 属性     | 说明                                                     |
| -------- | -------------------------------------------------------- |
| 物理类型 | 动态刚体（dynamic body）                                 |
| 受力     | 受重力影响，受碰撞力影响                                 |
| 数量     | 同一场景可存在多个小球                                   |
| 播放行为 | 唯一在播放态中运动的实体类型                             |
| 选中机制 | 可被点击选中为"当前选中球"，用于播放启动时的跟随判定     |

**关键规则**：在整个实体系统中，只有小球在播放态会被物理引擎推进。方块和音乐方块始终静止。

### 3.2 方块（Block）

| 属性     | 说明                                         |
| -------- | -------------------------------------------- |
| 物理类型 | 静态刚体（static body）                      |
| 用途     | 搭建轨道、斜坡、边界、平台等结构             |
| 播放行为 | 不移动，仅作为碰撞边界                       |
| 可编辑   | 位置、尺寸、旋转角度（均在编辑态设置）       |

### 3.3 音乐方块（Music Block）

| 属性     | 说明                                         |
| -------- | -------------------------------------------- |
| 物理类型 | 静态刚体（static body）                      |
| 用途     | 被小球碰撞时触发音符发声                     |
| 播放行为 | 不移动，检测碰撞并触发音频事件               |

**音乐参数**：

| 参数          | 类型   | 范围/约束     | 说明                                         |
| ------------- | ------ | ------------- | -------------------------------------------- |
| `noteName`    | string | 如 `C4`, `A3` | 音名，决定音高                               |
| `volume`      | number | 0 ~ 1         | 触发时的初始音量，同时决定自然衰减时长        |
| `timbre`      | string | 固定 `piano`  | v1 锁定钢琴音色，不可更改                    |

**无 `durationMs` 参数**：声音长度不由用户显式设置。触发后音符以初始音量开始发声，随后按音量自然衰减至静音。音量越高，衰减时间越长；音量越低，衰减时间越短。

## 4. 预测系统（Timeline 五线谱 + 预测线）

### 设计理念

预测系统是编辑态的核心辅助工具。它基于当前场景快照运行一次模拟预测（与播放使用相同的物理引擎），将结果分为两个维度呈现：

| 维度             | 呈现形式               | 回答的问题                 |
| ---------------- | ---------------------- | -------------------------- |
| 空间维度（预测线）| 画布上的轨迹虚线       | 小球会去哪里？会碰到什么？ |
| 音乐维度（Timeline）| 底部五线谱           | 什么时候触发什么音？       |

两者共享同一次预测模拟结果，保证完全一致。

### 预测触发时机

场景中任何影响物理模拟结果的变更都会触发预测重新计算：

- 新增、移动、删除任何实体
- 修改音乐方块的音名或音量
- 修改场景重力参数

### Timeline 五线谱

- **每个小球对应一条独立的五线谱**，以小球标识（颜色/编号）区分
- 横轴为预测时间，纵轴为音高（五线谱标准音高映射）
- 每个预测碰撞事件在对应小球的谱线上绘制一个音符
- 音符位置由预测碰撞时间（横轴）和音乐方块的 `noteName`（纵轴音高）决定
- 音符大小或透明度可反映 `volume` 参数

### PredictedNote 数据结构

预测模拟产生的每个碰撞事件记录为：

| 字段           | 类型   | 说明                         |
| -------------- | ------ | ---------------------------- |
| `timeMs`       | number | 预测碰撞时间（模拟开始后的毫秒）|
| `ballId`       | string | 碰撞的小球 ID                |
| `musicBlockId` | string | 被碰撞的音乐方块 ID          |
| `noteName`     | string | 该音乐方块的音名             |
| `volume`       | number | 该音乐方块的音量             |

**注意**：PredictedNote 是运行时计算结果，不进入持久化存储。

### Timeline 行为规则

| 场景         | Timeline UI   | 预测计算     |
| ------------ | ------------- | ------------ |
| 编辑模式     | 可见（五线谱）| 场景变更时重算 |
| 播放模式     | 隐藏          | 不计算       |
| 播放 -> 编辑 | 恢复可见      | 触发一次重算 |

## 5. 播放态触发系统

播放态中，碰撞触发是实时的，与预测系统无关：

```
小球物理运动 ──> 碰撞音乐方块 ──> AudioEngine 创建 voice ──> 发声
```

- 播放态不依赖预测数据，直接由物理引擎的实时碰撞事件驱动音频
- 每次碰撞创建独立的音符实例（voice），音量自然衰减
- 允许重叠发声，无冷却抑制

## 6. 跟随与镜头系统

### 跟随判定逻辑

```
按下 Space 启动播放
       │
       ▼
  有选中球？ ──Yes──> 相机自动跟随该球（平滑插值）
       │
      No
       │
       ▼
  相机保持当前位置（不自动跟随）
```

### 跟随规则

| 规则                   | 说明                                                       |
| ---------------------- | ---------------------------------------------------------- |
| 选中时机               | 编辑态点击小球即选中，不是跟随                             |
| 跟随启动               | 仅在按 `Space` 启动播放的那一刻判定，播放中不再重新判定     |
| 跟随方式               | 平滑插值移动相机中心到小球位置，避免硬锁定和跳变           |
| 手动覆盖               | 播放期间用户可随时手动平移/缩放，不受跟随限制              |
| 选中状态保留           | 停止播放返回编辑后，选中状态保留，直到用户显式改选或清空   |

## 7. 规则一致性汇总

以下规则贯穿整个系统设计，在实现中必须严格遵守：

| 编号 | 规则                                                         | 对应约束 |
| ---- | ------------------------------------------------------------ | -------- |
| R1   | `Space` 仅提供播放/停止，不存在暂停态                        | C1       |
| R2   | Timeline 是编辑态实时预测工具（五线谱），播放态隐藏且不计算  | C2       |
| R3   | 预测线仅编辑态可见                                           | C3       |
| R4   | 音乐触发允许重叠发声，无冷却抑制，音量驱动自然衰减           | C4       |
| R5   | 跟随基于"选中球 + 播放启动时判定"                            | C5       |
| R6   | 播放态禁止一切编辑操作                                       | --       |
| R7   | 仅小球在播放态中运动                                         | --       |
| R8   | Timeline 与预测线同源，基于同一次模拟预测                    | C2, C3   |

# 核心循环与玩法系统设计

## 1. 核心循环

1. 编辑模式选择积木并搭建场景。
2. 配置音乐方块参数（音名/音量/音长）。
3. 启动播放（`Space`），进入物理模拟与触发发声。
4. 停止播放（`Space` 或 `Esc`）回到编辑模式。
5. 观察 Timeline 历史事件并调整布局，进入下一轮。

## 2. 模式系统

### 编辑模式

- 可新增、选中、拖拽、删除三种积木。
- 可配置音乐方块参数。
- 可见右侧积木面板与底部 Timeline。
- 预测线可见。

### 播放模式

- 禁止任何场景编辑动作。
- 隐藏右侧积木面板与底部 Timeline。
- 预测线不可见。
- 允许视窗手动平移/缩放。

## 3. 实体系统

### 小球（Ball）

- 物理动态实体，受重力和碰撞影响。
- 同场景可存在多个小球。
- 可被选中为“当前选中球”，用于播放启动时跟随决策。

### 方块（Block）

- 静态碰撞实体。
- 用于搭建轨道、边界和结构。
- 播放中不移动。

### 音乐方块（Music Block）

- 静态触发实体。
- 小球碰撞后触发声音事件。
- 参数：`noteName`（音名）、`volume`（音量）、`durationMs`（音长）、`timbre=piano`（固定钢琴音色）。

## 4. 触发与时序系统

- 触发源：小球与音乐方块碰撞事件。
- 每次触发生成一条 `ContactEvent`，写入 Timeline 数据流。
- 事件记录最小字段：时间戳、小球 ID、音乐方块 ID、音名、音量。
- Timeline 仅在编辑模式展示，用于“上一轮播放结果回看”。

## 5. 跟随与镜头系统

- 播放前：若多球场景下用户选中了某个小球，启动播放后相机跟随该球。
- 无选中球：播放时不自动跟随（镜头保持当前用户视图，可手动操作）。
- 跟随方式：平滑插值（避免硬锁定和阶跃跳变）。

## 6. 规则一致性（本版关键）

- `Space`：编辑态开始播放；播放态停止并返回编辑态。
- 不存在“暂停态”。
- 播放态 Timeline 隐藏但持续记录数据。
- 预测线仅编辑态可见。
- 音乐触发可重叠，不做冷却抑制。

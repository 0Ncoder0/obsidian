# 02 -- 核心循环与玩法系统设计

## 1. 核心游戏循环

玩家的体验围绕一个不断迭代的循环展开：

```
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  ▼                                                  │
搭建场景 ──> 配置音乐参数 ──> 播放观察 ──> 停止回看 ──> 调整优化
 (编辑态)      (编辑态)       (播放态)     (编辑态)     (编辑态)
```

### 各阶段详细说明

| 阶段             | 模式   | 玩家行为                                           |
| ---------------- | ------ | -------------------------------------------------- |
| 1. 搭建场景      | 编辑   | 从右侧面板选择积木，在画布上放置和拖拽排列          |
| 2. 配置音乐参数  | 编辑   | 选中音乐方块，调整音名、音量、音长                  |
| 3. 播放观察      | 播放   | 按 `Space` 启动，观察弹珠运动和碰撞发声             |
| 4. 停止回看      | 编辑   | 按 `Space` 或 `Esc` 停止，查看 Timeline 中的事件记录 |
| 5. 调整优化      | 编辑   | 根据回看结果移动积木、修改参数，进入下一轮           |

---

## 2. 模式系统

游戏在任何时刻处于且仅处于两种模式之一。不存在暂停态或其他中间状态。

### 编辑模式（Edit）

玩家拥有完整的场景操作权限：

- 新增积木：从右侧面板选择类型，点击画布放置
- 选中积木：点击画布上的实体
- 移动积木：拖拽已选中的实体
- 删除积木：选中后按 `Delete` 或 `Backspace`
- 配置参数：选中音乐方块后，在右侧面板编辑参数
- 查看 Timeline：底部区域显示上一轮播放的事件记录
- 查看预测线：小球的预测运动轨迹可见

### 播放模式（Play）

场景完全锁定，物理模拟和音频系统接管：

- 所有编辑操作被禁止（新增、移动、删除、参数修改）
- 右侧编辑面板和底部 Timeline 隐藏
- 预测线隐藏
- 物理引擎驱动小球运动
- 碰撞事件触发音符发声
- Timeline 在后台持续记录事件（UI 不可见但数据在写入）
- 用户仅可进行视窗操作：平移和缩放

### 模式切换规则

```
              Space
  编辑模式 ◄────────► 播放模式
              Space
              Esc ──► 编辑模式（单向）
```

| 触发     | 当前模式 | 目标模式 | 附加行为                                     |
| -------- | -------- | -------- | -------------------------------------------- |
| `Space`  | 编辑     | 播放     | 锁编辑、隐藏 UI、判定跟随目标、启动物理/音频 |
| `Space`  | 播放     | 编辑     | 停止物理/音频、恢复 UI、展示 Timeline         |
| `Esc`    | 播放     | 编辑     | 同上                                         |
| `Esc`    | 编辑     | --       | 取消当前临时放置态（若存在），不切换模式      |

## 3. 实体系统

### 3.1 小球（Ball）

| 属性     | 说明                                                     |
| -------- | -------------------------------------------------------- |
| 物理类型 | 动态刚体（dynamic body）                                 |
| 受力     | 受重力影响，受碰撞力影响                                 |
| 数量     | 同一场景可存在多个小球                                   |
| 播放行为 | 唯一在播放态中运动的实体类型                             |
| 选中机制 | 可被点击选中为"当前选中球"，用于播放启动时的跟随判定     |

**关键规则**：在整个实体系统中，只有小球在播放态会被物理引擎推进。方块和音乐方块始终静止。

### 3.2 方块（Block）

| 属性     | 说明                                         |
| -------- | -------------------------------------------- |
| 物理类型 | 静态刚体（static body）                      |
| 用途     | 搭建轨道、斜坡、边界、平台等结构             |
| 播放行为 | 不移动，仅作为碰撞边界                       |
| 可编辑   | 位置、尺寸、旋转角度（均在编辑态设置）       |

### 3.3 音乐方块（Music Block）

| 属性     | 说明                                         |
| -------- | -------------------------------------------- |
| 物理类型 | 静态刚体（static body）                      |
| 用途     | 被小球碰撞时触发音符发声                     |
| 播放行为 | 不移动，检测碰撞并触发音频事件               |

**音乐参数**：

| 参数          | 类型   | 范围/约束     | 说明                     |
| ------------- | ------ | ------------- | ------------------------ |
| `noteName`    | string | 如 `C4`, `A3` | 音名，决定音高           |
| `volume`      | number | 0 ~ 1         | 触发时的初始音量         |
| `durationMs`  | number | > 0           | 包络持续时长（毫秒）     |
| `timbre`      | string | 固定 `piano`  | v1 锁定钢琴音色，不可更改 |

## 4. 触发与时序系统

### 触发流程

```
小球运动 ──> 碰撞音乐方块 ──> 生成 ContactEvent ──> 触发音符发声
                                     │
                                     ▼
                              写入 Timeline 数据流
```

### ContactEvent 数据结构

每次碰撞生成一条事件记录，包含以下最小字段：

| 字段           | 类型   | 说明                       |
| -------------- | ------ | -------------------------- |
| `id`           | string | 事件唯一标识               |
| `timeMs`       | number | 触发时间（播放开始后的毫秒）|
| `ballId`       | string | 触发碰撞的小球 ID          |
| `musicBlockId` | string | 被碰撞的音乐方块 ID        |
| `noteName`     | string | 该音乐方块的音名           |
| `volume`       | number | 该音乐方块的音量           |

### Timeline 行为规则

| 场景         | Timeline UI | Timeline 数据记录 |
| ------------ | ----------- | ----------------- |
| 编辑模式     | 可见（展示上一轮事件） | 不写入新数据 |
| 播放模式     | 隐藏        | 持续写入          |
| 播放 -> 编辑 | 恢复可见    | 展示本轮记录      |

## 5. 跟随与镜头系统

### 跟随判定逻辑

```
按下 Space 启动播放
       │
       ▼
  有选中球？ ──Yes──> 相机自动跟随该球（平滑插值）
       │
      No
       │
       ▼
  相机保持当前位置（不自动跟随）
```

### 跟随规则

| 规则                   | 说明                                                       |
| ---------------------- | ---------------------------------------------------------- |
| 选中时机               | 编辑态点击小球即选中，不是跟随                             |
| 跟随启动               | 仅在按 `Space` 启动播放的那一刻判定，播放中不再重新判定     |
| 跟随方式               | 平滑插值移动相机中心到小球位置，避免硬锁定和跳变           |
| 手动覆盖               | 播放期间用户可随时手动平移/缩放，不受跟随限制              |
| 选中状态保留           | 停止播放返回编辑后，选中状态保留，直到用户显式改选或清空   |

## 6. 规则一致性汇总

以下规则贯穿整个系统设计，在实现中必须严格遵守：

| 编号 | 规则                                     | 对应约束 |
| ---- | ---------------------------------------- | -------- |
| R1   | `Space` 仅提供播放/停止，不存在暂停态    | C1       |
| R2   | 播放态 Timeline 隐藏但数据持续记录       | C2       |
| R3   | 预测线仅编辑态可见                       | C3       |
| R4   | 音乐触发允许重叠发声，无冷却抑制         | C4       |
| R5   | 跟随基于"选中球 + 播放启动时判定"        | C5       |
| R6   | 播放态禁止一切编辑操作                   | --       |
| R7   | 仅小球在播放态中运动                     | --       |

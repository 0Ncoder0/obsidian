## 2. 技术风险矩阵

| 风险 ID | 风险描述                                 | 可能性 | 影响 | 综合 |
| ------- | ---------------------------------------- | ------ | ---- | ---- |
| TR-01   | 高频碰撞导致音频实例激增，产生爆音或卡顿 | 高     | 高   | 严重 |
| TR-02   | 浏览器音频策略差异导致首帧触发不稳定     | 中     | 中   | 中等 |
| TR-03   | localStorage 配额或权限导致自动保存失败  | 中     | 高   | 中等 |
| TR-04   | 复杂场景下相机跟随与手动操作冲突         | 低     | 中   | 低   |
| TR-05   | Canvas 2D 在大量实体时渲染性能不足       | 低     | 中   | 低   |

## 3. 风险缓解策略

### TR-01：音频实例激增

| 措施           | 说明                                                     |
| -------------- | -------------------------------------------------------- |
| 同帧 voice 上限 | 单帧最多创建 N 个新 voice（建议 N = 8~16）              |
| 总活跃上限     | 全局同时存在的 voice 不超过 M 个（建议 M = 32~64）       |
| 诊断日志       | 超限时记录日志，便于定位极端碰撞场景                     |
| 监控面板       | 开发模式下实时显示活跃 voice 数                          |

### TR-02：浏览器音频策略

| 措施              | 说明                                                   |
| ----------------- | ------------------------------------------------------ |
| 显式初始化        | 首次用户交互后调用 `AudioContext.resume()`              |
| 状态检测          | 监听 AudioContext 状态变化，suspended 时提示用户        |
| 静音降级          | 音频不可用时物理播放可继续（静音模式）                  |

### TR-03：存储失败

| 措施              | 说明                                                   |
| ----------------- | ------------------------------------------------------ |
| 失败提示          | 保存失败时 HUD 显示明确提示和可行动建议                |
| 重试机制          | 保存失败后下一次编辑操作时重试                         |
| 容量预检          | 可选：在保存前估算数据大小，接近配额时提前告警         |

### TR-04：相机冲突

| 措施              | 说明                                                   |
| ----------------- | ------------------------------------------------------ |
| 优先级规则        | 用户手动操作优先级高于自动跟随                         |
| 跟随恢复          | 用户手动操作后一段时间不操作，不自动恢复跟随           |

## 4. 文档维护策略

### 变更流程

1. 任何规则变更，首先更新 `00-index.md` 的核心设计约束表
2. 同步更新受影响的分卷文档（根据下表确定范围）
3. 发布前执行跨文档一致性扫描

### 分卷影响矩阵

当以下关键概念发生变更时，需要同步更新对应的分卷：

| 变更领域     | 必须更新的分卷                      |
| ------------ | ----------------------------------- |
| 输入映射变更 | 03（交互）、04（UI）、08（测试）    |
| 模式规则变更 | 02（核心）、03（交互）、06（架构）、08（测试） |
| 音频行为变更 | 05（音频）、06（架构）、08（测试）  |
| 数据模型变更 | 07（数据）、06（架构）、08（测试）  |
| UI 布局变更  | 04（UI）、03（交互）                |

### 一致性扫描关键词

发布前搜索以下关键词，确保各分卷描述一致：

`Space`、`Timeline`、`预测线`、`跟随`、`重叠发声`、`编辑态`、`播放态`、`暂停`

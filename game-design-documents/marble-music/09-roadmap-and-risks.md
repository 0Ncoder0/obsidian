# 演进路线与风险管理

## 1. v1 后续演进方向

- 音乐层：
  - 多音色扩展（如马林巴、合成 Pad）。
  - 更完整包络和效果器（混响、滤波）。
- 交互层：
  - Timeline 事件筛选与定位跳转。
  - 预测线细节开关（采样密度、颜色映射）。
- 内容层：
  - 预置场景模板。
  - 教学引导与挑战关卡。

## 2. 技术风险

- 高频碰撞导致音频实例激增，可能带来性能或爆音风险。
- 浏览器音频策略差异导致首帧触发不稳定。
- localStorage 配额或权限导致自动保存失败。
- 复杂场景中相机跟随和平移操作的冲突体验。

## 3. 风险缓解策略

- 为活跃 voice 数设置软上限与诊断日志。
- 首次交互后显式初始化音频上下文。
- 保存失败提供 UI 提示并保留重试机会。
- 跟随状态与手动相机操作定义优先级，避免抢控制权。

## 4. 文档维护策略

- 每次规则变更先更新 `00-index.md` 的“最终规则”。
- 任何影响输入、UI、技术行为的变更，必须同时更新 `03/04/06/08` 四卷。
- 发布前执行一次跨文档一致性扫描（关键词：Space、Timeline、预测线、跟随、重叠发声）。

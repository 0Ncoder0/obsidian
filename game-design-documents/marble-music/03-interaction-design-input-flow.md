# 交互设计与输入流程

## 1. 输入设备与上下文

- 输入设备：鼠标 + 键盘（PC Web）。
- 输入解释依赖模式：编辑模式与播放模式共享部分全局操作，但编辑权限不同。

## 2. 全局输入映射

- 鼠标滚轮：缩放视窗。
- 中键拖拽（或 `Alt+左键拖拽`）：平移视窗。
- `Esc`：
  - 播放模式：停止播放并回到编辑模式。
  - 编辑模式：取消当前临时放置态（若存在）。
- `Space`：
  - 编辑模式：开始播放。
  - 播放模式：停止播放并回到编辑模式。

## 3. 编辑模式输入

- 左键点击右侧积木按钮：切换当前放置类型（小球/方块/音乐方块）。
- 左键点击画布空白：放置当前类型积木。
- 左键点击实体：选中实体。
- 左键拖拽实体：移动实体（编辑模式限定）。
- `Delete` / `Backspace`：删除当前选中实体。
- `1` / `2` / `3`：快捷切换积木类型。

## 4. 多球选中与跟随决策（定稿）

- 当场景中有多个小球时，点击某个小球仅表示“选中球”，不是立即进入跟随。
- 在编辑模式按 `Space` 启动播放时：
  - 若存在选中球：播放开始即自动跟随该球。
  - 若不存在选中球：播放开始后不自动跟随。
- 播放期间用户仍可手动平移/缩放相机。
- 停止播放返回编辑后，选中状态保留到下一次显式改选或清空。

## 5. 音乐方块参数编辑交互

- 选中音乐方块后，右侧参数面板显示：
  - `noteName`：音名（如 `C4`）。
  - `volume`：0~1。
  - `durationMs`：毫秒，>0。
- 参数实时生效并进入自动保存队列。
- 播放模式下参数面板不可编辑（并随右侧面板整体隐藏）。

## 6. 模式切换流程

### 编辑 -> 播放

1. 锁定编辑输入。
2. 隐藏右侧积木区与底部 Timeline。
3. 根据“选中球”决定是否启用自动跟随。
4. 启动物理和音频系统。

### 播放 -> 编辑

1. 停止物理推进与发声触发。
2. 恢复编辑输入权限。
3. 恢复右侧积木区与底部 Timeline。
4. Timeline 展示刚结束播放会话事件。

## 7. 反馈规范

- 选中反馈：实体高亮轮廓 + 参数面板上下文同步。
- 不可操作反馈：播放态对编辑尝试给出轻提示或直接吞掉事件（实现保持一致）。
- 保存反馈：自动保存成功/失败有状态提示（见 UI 文档）。

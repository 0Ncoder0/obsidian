# UI/UX 与视觉设计规范（v1）

## 1. 布局总览

- 中央：主画布（物理沙盒与实体渲染）。
- 右侧：编辑面板（积木选择 + 参数编辑）。
- 底部：Timeline（触碰事件回看）。
- 顶部/角落：播放状态与保存状态轻量 HUD。

## 2. 模式驱动可见性

### 编辑模式

- 右侧编辑面板：显示。
- 底部 Timeline：显示。
- 预测线：显示。
- 播放控制：显示“开始播放”主动作。

### 播放模式

- 右侧编辑面板：隐藏。
- 底部 Timeline：隐藏。
- 预测线：隐藏。
- 播放控制：显示“停止播放”主动作。

## 3. 画布视觉层级

从下到上：
1. 背景网格/空间参考层。
2. 静态方块层。
3. 音乐方块层（带可识别音高/状态标记）。
4. 小球层（主视觉动态对象）。
5. 选中态与高亮层。
6. 可视化特效层（发声脉冲、频谱/强度反馈）。

## 4. 关键组件规范

### 积木选择器

- 三个按钮固定顺序：小球、方块、音乐方块。
- 当前选择态高亮。
- 与快捷键 `1/2/3` 一一对应。

### 音乐参数面板

- 仅在选中音乐方块时展示参数区。
- 字段输入约束可见（范围、格式、单位）。
- 非法输入即时提示，不允许写入非法值。

### Timeline（编辑态）

- 按时间从左到右或从上到下展示事件。
- 每条事件至少包含：触发时间、音名、来源球与方块标识。
- 支持滚动查看（不要求 v1 提供复杂筛选）。

### 保存状态 Banner

- 状态最小集合：保存中、已保存、保存失败。
- 保存失败时需要给出可行动提示（如“请检查浏览器存储权限”）。

## 5. 动效与过渡

- 模式切换：面板隐藏/显示采用短时淡入淡出或滑动过渡。
- 跟随相机：平滑移动，禁止瞬移抖动。
- 发声可视化：与音量和活动状态同步变化，确保反馈可感知。

## 6. 可用性约束

- 播放模式绝不出现可误导的编辑入口。
- 隐藏状态下组件不响应编辑事件。
- 视觉强调“当前模式”，避免用户误判是否可编辑。

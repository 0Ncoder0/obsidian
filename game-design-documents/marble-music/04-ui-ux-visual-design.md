# 04 -- UI/UX 与视觉设计规范

## 1. 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  [播放状态 HUD]                        [保存状态 HUD]   │
│                                                         │
│                                          ┌────────────┐ │
│                                          │ 积木选择器 │ │
│                                          │ ┌────────┐ │ │
│              主画布                      │ │ ● 小球  │ │ │
│         （物理沙盒 Canvas）              │ │ ■ 方块  │ │ │
│                                          │ │ ♪ 音乐  │ │ │
│                                          │ └────────┘ │ │
│                                          │            │ │
│                                          │ 参数面板   │ │
│                                          │ (选中音乐  │ │
│                                          │  方块时)   │ │
│                                          └────────────┘ │
│                                                         │
├─────────────────────────────────────────────────────────┤
│              Timeline（碰撞事件回看）                    │
│  [00:00.1 C4 ●→♪]  [00:00.3 E4 ●→♪]  [00:01.2 G4 ●→♪] │
└─────────────────────────────────────────────────────────┘
```

### 区域说明

| 区域           | 位置   | 功能                                             |
| -------------- | ------ | ------------------------------------------------ |
| 主画布         | 中央   | 物理沙盒渲染，积木摆放和物理模拟的主视图         |
| 积木选择器     | 右侧上 | 三个积木类型按钮，用于选择当前放置类型           |
| 参数面板       | 右侧下 | 选中音乐方块时显示参数编辑（音名/音量/音长）     |
| Timeline       | 底部   | 上一轮播放的碰撞事件按时间排列                   |
| 播放状态 HUD   | 左上角 | 显示当前模式（编辑/播放）和播放控制提示           |
| 保存状态 HUD   | 右上角 | 显示自动保存状态（保存中/已保存/保存失败）       |

---

## 2. 模式驱动的 UI 可见性

| 组件            | 编辑模式 | 播放模式 |
| --------------- | -------- | -------- |
| 右侧积木选择器  | 显示     | **隐藏** |
| 右侧参数面板    | 显示     | **隐藏** |
| 底部 Timeline   | 显示     | **隐藏** |
| 预测线          | 显示     | **隐藏** |
| 播放状态 HUD    | 显示"按 Space 播放" | 显示"播放中...按 Space 停止" |
| 保存状态 HUD    | 显示     | 显示     |
| 音乐活动可视化  | 不活跃   | **活跃** |

**关键原则**：播放模式下不能出现任何可能误导玩家进行编辑的 UI 元素。隐藏的组件不响应任何事件。

## 3. 画布视觉层级

从下到上的渲染顺序（z-index 由低到高）：

| 层级 | 内容                 | 说明                                       |
| ---- | -------------------- | ------------------------------------------ |
| L0   | 背景网格             | 空间参考线，帮助玩家判断距离和对齐         |
| L1   | 静态方块             | 普通方块，基础结构色                       |
| L2   | 音乐方块             | 带音高标记（如显示"C4"）和状态指示色       |
| L3   | 小球                 | 主视觉动态对象，高辨识度颜色               |
| L4   | 选中高亮层           | 选中实体的轮廓高亮叠加                     |
| L5   | 预测线层             | 编辑态小球轨迹预览线（半透明虚线）         |
| L6   | 可视化特效层         | 发声脉冲环、能量反馈等播放态叠加效果       |

## 4. 关键组件规范

### 4.1 积木选择器

```
┌────────────┐
│  ● 小球 [1] │  ← 当前选中时高亮背景
├────────────┤
│  ■ 方块 [2] │
├────────────┤
│  ♪ 音乐 [3] │
└────────────┘
```

- 三个按钮固定顺序，不可重排
- 当前选中类型高亮显示（背景色变化或边框强调）
- 每个按钮旁显示对应快捷键（`1` / `2` / `3`）
- 按钮尺寸足够大，易于点击

### 4.2 音乐方块参数面板

仅在编辑模式下选中音乐方块时显示：

```
┌──────────────────┐
│  ♪ 音乐方块参数   │
├──────────────────┤
│  音名  [C4    ▼] │  ← 下拉选择或文本输入
│                  │
│  音量  ■■■■□□  0.7│  ← 滑块 + 数值显示
│                  │
│  音长  [500] ms  │  ← 数字输入 + 单位标注
└──────────────────┘
```

- 字段输入约束可见：范围、格式、单位均标注
- 非法输入即时提示（红色边框 + 错误文字），不允许写入非法值
- 参数修改实时生效
- 未选中音乐方块时，此区域不显示或显示占位提示

### 4.3 Timeline

编辑态底部区域，展示上一轮播放的碰撞事件：

```
┌───────────────────────────────────────────────────────┐
│ Timeline                                    ← 可滚动 │
│                                                       │
│  00:00.120  C4  Ball-1 → MusicBlock-3   ■■■■          │
│  00:00.350  E4  Ball-1 → MusicBlock-5   ■■■           │
│  00:01.200  G4  Ball-2 → MusicBlock-1   ■■■■■         │
│  00:02.100  C5  Ball-1 → MusicBlock-7   ■■            │
│                                                       │
└───────────────────────────────────────────────────────┘
```

- 按时间从上到下（或从左到右）排列
- 每条事件至少显示：触发时间、音名、来源小球标识、目标音乐方块标识
- 可选：音量用条形图或点阵直观表示
- 支持滚动查看（v1 不要求筛选或跳转功能）
- 无事件时显示空状态提示

### 4.4 保存状态 HUD

```
  保存中...  │  已保存 ✓  │  保存失败 ✗ 请检查浏览器存储权限
```

- 三种状态：保存中、已保存、保存失败
- 保存成功后短暂显示后自动淡出
- 保存失败时持续显示，附可行动提示文案

## 5. 动效与过渡

| 场景               | 动效                                 | 时长建议     |
| ------------------ | ------------------------------------ | ------------ |
| 模式切换面板隐藏   | 右侧面板向右滑出 + 淡出             | 200 ~ 300 ms |
| 模式切换面板显示   | 右侧面板从右滑入 + 淡入             | 200 ~ 300 ms |
| Timeline 隐藏/显示 | 底部面板向下滑出/从下滑入            | 200 ~ 300 ms |
| 相机跟随           | 平滑插值移动，禁止瞬移和阶跃跳变    | 持续性       |
| 发声可视化脉冲     | 音乐方块位置扩散环，随音量缩放亮度   | 与音长同步   |
| 发声结束衰退       | 脉冲环平滑缩小 + 淡出               | 跟随包络衰减 |
| 实体选中           | 高亮轮廓淡入                         | 100 ~ 150 ms |

## 6. 可用性约束

| 约束编号 | 规则                                                       |
| -------- | ---------------------------------------------------------- |
| U1       | 播放模式绝不出现可误导的编辑入口                           |
| U2       | 隐藏状态下的组件不响应任何编辑事件                         |
| U3       | 当前模式有明确的视觉强调，玩家不应误判是否可编辑           |
| U4       | 所有可交互元素有明确的 hover/active 反馈                   |
| U5       | 参数输入约束在 UI 层可见，不需要玩家记忆有效范围           |
| U6       | 保存失败必须给出可行动提示，不静默丢失                     |

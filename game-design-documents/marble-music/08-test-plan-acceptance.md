# 08 -- 测试计划与验收标准

## 1. 测试目标

- 验证核心玩法闭环（搭建 -> 播放 -> 回看 -> 调整）稳定可用
- 验证模式隔离严格，播放态无法进行任何编辑操作
- 验证五条核心约束（C1~C5）在所有场景下得到遵守
- 验证存档恢复的可靠性和数据完整性

---

## 2. 测试分层

每条核心约束至少对应 1 条单元测试 + 1 条端到端测试：

| 约束 | 规则描述                                         | 单元测试                | E2E 测试            |
| ---- | ------------------------------------------------ | ----------------------- | ------------------- |
| C1   | Space 仅播放/停止，无暂停                        | MC-02, MC-03            | E2E-07 ~ E2E-10    |
| C2   | 播放态 Timeline 隐藏，数据持续记录               | TR-01, TR-02            | E2E-06, E2E-02     |
| C3   | 预测线仅编辑态显示                               | PE-01, PE-02            | E2E-06             |
| C4   | 音乐触发允许重叠发声                             | AE-01, AE-02            | E2E-01             |
| C5   | 选中球跟随策略                                   | --                      | E2E-11 ~ E2E-13    |

## 4. 缺陷优先级

| 等级 | 定义                         | 示例                                       |
| ---- | ---------------------------- | ------------------------------------------ |
| P0   | 阻断核心体验或数据安全       | 模式切换崩溃、编辑锁失效、存档损坏         |
| P1   | 核心功能异常但不崩溃         | 发声错误、跟随策略错误、Timeline 记录丢失  |
| P2   | 体验或视觉细节问题           | 动画不流畅、文案不一致、辅助 UI 显示异常   |

---

## 5. 发布门禁（v1）

发布前必须满足以下全部条件：

| 门禁条件                     | 验证方式                       |
| ---------------------------- | ------------------------------ |
| 所有 E2E 用例通过            | Playwright 测试报告            |
| 所有单元测试通过             | Vitest 测试报告                |
| 无未解决的 P0 缺陷           | 缺陷跟踪系统                  |
| 核心约束 C1~C5 全部验证通过  | 验收矩阵逐项确认              |
| 标准场景下无明显性能问题     | 手动验证（20 积木 + 5 小球）   |

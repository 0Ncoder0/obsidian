# 08 -- 测试计划与验收标准

## 1. 测试目标

- 验证核心玩法闭环（搭建 -> 预览五线谱 -> 播放验证 -> 调整）稳定可用
- 验证模式隔离严格，播放态无法进行任何编辑操作
- 验证五条核心约束（C1~C5）在所有场景下得到遵守
- 验证 Timeline 五线谱与预测线保持数据一致性
- 验证存档恢复的可靠性和数据完整性

## 4. 缺陷优先级

| 等级 | 定义                         | 示例                                                       |
| ---- | ---------------------------- | ---------------------------------------------------------- |
| P0   | 阻断核心体验或数据安全       | 模式切换崩溃、编辑锁失效、存档损坏                         |
| P1   | 核心功能异常但不崩溃         | 发声错误、跟随策略错误、五线谱预测与实际播放不一致         |
| P2   | 体验或视觉细节问题           | 动画不流畅、文案不一致、五线谱音符显示位置微偏             |

## 5. 发布门禁（v1）

发布前必须满足以下全部条件：

| 门禁条件                     | 验证方式                       |
| ---------------------------- | ------------------------------ |
| 所有 E2E 用例通过            | Playwright 测试报告            |
| 所有单元测试通过             | Vitest 测试报告                |
| 无未解决的 P0 缺陷           | 缺陷跟踪系统                  |
| 核心约束 C1~C5 全部验证通过  | 验收矩阵逐项确认              |
| 标准场景下无明显性能问题     | 手动验证（20 积木 + 5 小球）   |
| Timeline 五线谱预测与播放结果一致 | 手动对比验证               |

# 测试计划与验收标准

## 1. 测试目标

- 验证核心玩法闭环稳定可用。
- 验证模式隔离不会产生误编辑。
- 验证音频触发、跟随策略、存档恢复满足定稿规则。

## 2. 测试分层

### 单元测试（Vitest）

- `ModeController`：状态切换、权限门禁。
- `PhysicsWorld`：动静态规则（仅球移动）。
- `AudioEngine`：触发参数映射、可重叠发声调度。
- `PredictionEngine`：仅编辑态有效。
- `TimelineRecorder`：事件顺序和字段完整性。
- `LocalSaveRepository`：节流保存、恢复与异常处理。

### 端到端测试（Playwright）

- 搭建并触发发声闭环。
- 播放态 UI 隐藏规则。
- `Space` 播放/停止（无暂停）行为。
- 多球下“选中球 -> 启动播放自动跟随”。
- 无选中球时播放不自动跟随。
- 刷新后恢复场景与音乐参数。

## 3. 验收矩阵（关键规则）

- 规则 R1：`Space` 不提供暂停，仅播放/停止。
- 规则 R2：播放态隐藏 Timeline，但事件持续记录。
- 规则 R3：预测线仅编辑态显示。
- 规则 R4：音乐触发允许重叠发声。
- 规则 R5：多球下点击仅选中；按 `Space` 后按选中球决定是否自动跟随。

每条规则至少应有：
- 1 条单元验证（逻辑层）
- 1 条 E2E 验证（真实交互层）

## 4. 回归优先级

- P0：模式切换和编辑锁、崩溃类问题、存档损坏。
- P1：触发发声错误、跟随策略错误、Timeline 记录丢失。
- P2：视觉细节、文案一致性、辅助 UI 体验。

## 5. 发布门禁（v1）

- 关键 E2E 用例全通过。
- 单元测试全通过。
- 无 P0 未解决缺陷。
- 无“规则 R1~R5”冲突项。
